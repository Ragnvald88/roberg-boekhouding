Dashboard Redesign: 3 Approaches

Which direction feels right for your boekhouding dashboard?

A

Command Center

Hero metrics on top, everything else earns its space

3 large hero cards (Omzet, Winst, Belasting) dominate the top. Below: a compact secondary strip for Uren, Km, Documenten — only when they have meaningful values. Alerts appear contextually. Charts get more room. Empty states are hidden, not shown as zeros.

Pros

  • Clear hierarchy — you instantly see what matters
  • Sparklines in hero cards show trends at a glance
  • Clean, professional, inspired by Xero/FreshBooks
  • Empty/zero metrics hidden → no visual noise

Cons

  • Some metrics only visible when relevant (intentional trade-off)
B

Financial Story

Top-down narrative: Earn → Spend → Keep → Act

Dashboard reads like a financial summary. Section 1: "Wat je verdient" (Omzet + trend chart integrated). Section 2: "Wat je uitgeeft" (Kosten + donut). Section 3: "Wat je houdt" (Winst + Belasting). Section 4: "Wat je moet doen" (alerts, openstaand, ongefactureerd). Each section is a visual block.

Pros

  • Tells a story — logical flow that makes sense
  • Charts are integrated with their KPIs, not separated
  • Alerts become a clear action section

Cons

  • Longer page — more scrolling needed
  • Less "at a glance" — more "read through"
  • More radical departure from current layout
C

Hybrid: Command Center + Smart Sections

Best of both — hero KPIs up top, sectioned content below

3 hero KPI cards with sparklines at the top (Omzet, Winst, Belasting). Below: two clear sections — "Overzicht" with the revenue chart + cost donut side by side, and "Aandachtspunten" with contextual alerts (ongefactureerd, openstaand, uren, documenten) that only appear when relevant. Secondary metrics (Km, Uren, Documenten) become a compact icon-strip, not full cards. Quick actions integrated into the header area.

Pros

  • Hero KPIs give instant pulse check
  • Sections create logical grouping without forcing a narrative
  • Contextual alerts = zero noise when things are fine
  • Compact secondary strip avoids card bloat
  • Still fits on one screen without excessive scrolling

Cons

  • Slightly more complex layout logic
