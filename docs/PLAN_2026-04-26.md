## A. Verified bugs (with severity)

### A1. P1 — Year-lock gap in werkdagen mutations (TWO call sites)
- `database.py:1273-1285` — `link_werkdagen_to_factuur` updates `werkdagen.factuurnummer` with no `assert_year_writable` on the werkdag dates.
- `database.py:1355-1361` — `save_factuur_atomic` has a *second* inline werkdag UPDATE that's also unguarded.
- **Exploit**: opening the invoice builder, selecting a werkdag from a definitief-locked year, and saving silently mutates the locked werkdag's `factuurnummer` field. Direct K6 violation.
- **Fix shape**: shared helper `_assert_werkdagen_writable(db_path, werkdag_ids)` called from both sites.

### A2. P1 — Bulk "Markeer als privé" bypasses factuur-link guard
- `pages/transacties.py:936-955` — `bulk_negeren()` iterates `table_ref['table'].selected` directly, no `koppeling_type='factuur'` filter, no snapshot.
- `database.py:1697-1701` — `mark_banktx_genegeerd` sets `genegeerd=1` blindly; only checks `assert_year_writable`.
- **Exploit**: bulk-select a bank-tx that's matched to a verstuurd factuur → flip to "privé" → factuur still claims matched, the bank link is now invisible from `/transacties`.
- **Fix shape**: defense-in-depth — DB-level check in `mark_banktx_genegeerd` (raise `ValueError` when setting genegeerd=1 on a factuur-linked row) **plus** UI-level skip in `bulk_negeren` for visible-feedback.

### A3. P1 (UX) — Health alerts have four concrete gaps
All in `database.py:get_health_alerts` (2591-2669):
- **A3a** No stale-werkdagen check (>30d old + `factuurnummer = ''`). Most-actionable signal for a working huisarts is missing.
- **A3b** Concept-invoice severity hard-coded `'info'` regardless of age (`database.py:2647`). >14d concept = forgotten work.
- **A3c** Overdue threshold is single 14d cutoff, no escalation (>30d → "stuur herinnering").
- **A3d** Stale `/bank` link at `database.py:2623` and `pages/jaarafsluiting.py:193`. `/bank` is now a redirect (`pages/bank.py:10-12`); both should point to `/transacties?status=ongecategoriseerd` and `/transacties` respectively.

### A4. P2 — Bulk-actions in `/facturen` use live `table.selected` reference, not snapshot
- `pages/facturen.py:1011, 1029, 1052, 1064` — `selected = table.selected` is a reference; the dialog shows a captured `nummers` list but the action loop uses the live ref.
- Same anti-pattern that's *already fixed* in `/transacties` (`pages/transacties.py:963` has the comment `rows_to_delete is the snapshot captured`).
- **Fix shape**: `selected = list(table.selected or [])` at dialog-open — copy the existing transacties pattern.
- **Why P2 not P1**: requires the user to actively change selection mid-dialog. Real but narrow.

### A5. P2 — Dashboard `has_va` ignores manually-entered ZVW VA
- `pages/dashboard.py:283-285` — checks `fp.voorlopige_aanslag_betaald` (IB-only) OR `va_data['has_bank_data']` (any BD bank match). Misses the case where user enters annual ZVW VA manually in `/instellingen` with no bank data yet.
- **Fix shape**: extract `_has_va_data(fp, va_data)` helper that also checks `voorlopige_aanslag_zvw_betaald`; one-line dashboard call site change + a unit test.

### A6. P2 — No global jaar state
- `components/layout.py` — no shared year context; every page declares its own `jaar_select` and resets to `date.today().year` on navigation.
- `app.storage.user` is currently used only for `selected_werkdagen` handoff (`pages/werkdagen.py:332/380`).
- **Fix shape**: `app.storage.user['jaar']`, layout owns the selector, pages subscribe. Touches 8 pages.

### A7. P3 — Box 3 2026 percentages are voorlopig but undifferentiated
- `import_/seed_data.py:180-181` seeds 2026 Box 3 as `1.28/6.00/2.70`. Per Belastingdienst, 2026 banktegoeden + schulden percentages remain *voorlopig* until ~Q1 2027.
- The CLAUDE.md rule "use DEFINITIEVE not voorlopig" applies to closed past years (2023-2025 are already correctly definitief). For the live year, voorlopig is the only option — system is doing the right thing computationally.
- **Fix shape**: pure labeling — seed comment says "voorlopig 2026, vervangen Q1 2027"; aangifte page shows a "voorlopige percentages" badge for the live year. No calculation change. Schedule a /schedule agent for Q1 2027 to bump definitief.

### Hidden dependencies (verified, must not be missed)

1. **Lane 1 has two call sites for werkdagen mutation** — `link_werkdagen_to_factuur` AND inline UPDATE in `save_factuur_atomic`. A guard helper covers both; fixing only one leaves a hole in the normal builder save path.
2. **Lane 1 `mark_banktx_genegeerd` strengthening must be paired** with the UI bulk_negeren change, otherwise the UI silently swallows a new `ValueError` that was never thrown before.
3. **Lane 2 risks duplicate dashboard surfaces** — dashboard already renders ongefactureerde werkdagen and openstaande facturen as their own AANDACHTSPUNTEN cards (`pages/dashboard.py:550-617`). Adding a stale-werkdagen alert in `get_health_alerts` will create a second card unless we either (a) skip the alert when the dedicated card already shows OR (b) collapse all into the alert system. Recommend (a): keep dedicated cards for "any" cases, alerts fire only on the *stale* (>30d) subset.
4. **Lane 2 severity escalation** — current dashboard severity map only handles `'info'` and `'warning'` (`pages/dashboard.py:623-632`). Adding `'critical'` requires a third style entry; otherwise it falls through to `info` styling and the escalation isn't visible.
5. **Lane 5 jaar-state** — pywebview native mode runs in a child process that re-imports `main.py`. Verify `app.storage.user` survives the pywebview child reimport (CLAUDE.md's "no top-level side-effects in main.py" note hints this is non-trivial). Pre-flight check before committing to the design.

   ```python
   async def _assert_werkdagen_writable(db_path: Path, werkdag_ids: list[int]) -> None:

       # Fetch datums for the given IDs, call assert_year_writable on each unique year.
   ```
